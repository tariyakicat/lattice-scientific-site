# Blog Authoring Guide

## Add a post

Create one MDX file in `src/content/blog/`. The filename becomes the public slug. Fill every required frontmatter field and use an existing author ID from `src/content/authors/`.

## Answer-first structure

- Phrase H2 headings as questions a researcher might search.
- Make the first sentence below every H2 a direct, standalone answer.
- Put the most useful definitions, decisions, and constraints in the top third.
- Keep paragraphs short. Prefer meaningful lists and semantic tables where comparison helps.
- Use `ComparisonTable`, `Callout`, and `Stat` rather than hand-built markup.
- Link statistics to the original or authoritative source. Do not cite an aggregator when a primary source exists.
- Add only claims that have been checked. Keep unfinished sections visibly framed as editorial placeholders.

## Images and comparisons

- Use descriptive image context in the article title and captions.
- Add `draftFinal` only when both images are cleared for public display.
- Keep dates honest: update `dateModified` whenever facts, recommendations, or examples materially change.

## Final review

- Exactly one H1 is generated by the post template.
- Meta description is unique and no longer than 155 characters.
- TL;DR is 40–60 words and answers the main query without marketing language.
- FAQ answers match the visible article and are useful without clicking elsewhere.
- Links, names, permissions, costs, and specifications are verified before publishing.
